Hello, Developer!


Welcome to the Bytengine API developer documentation! Here you'll find information to help you administer the server as well as save and retrieve data!

About Bytengine

Bytengine is a data server and application backend services provider. Data is stored in JSON and Binary format and can be accessed using Bytengine's REST API. Core server commands allow storage and retrieval of data. Bytengine is also designed to allow developers to extend it by implementing their own server commands.

Bytengine BQL

Bytengine Query Language (BQL) is a data search and retrieval mini language. Loosely based on Structured Query Language (SQL), BQL has fewer keywords/functions while remaining powerful and expressive enough to allow developers to build complex queries. BQL can also be extended to include user defined function.

Byterminal

Byterminal is Bytengine's web based administration console. From the terminal you can create and manage databases, users and content. The terminal is an example of the type of applications that can be built with Bytengine. Examples in the command help documentation can be typed directly in the terminal to interact with Bytengine.

Databases

A Bytengine server instance can contain multiple content storage databases. Databases constitute the first level of content organization. Having multiple databases allows Bytengine to be used as a single content repository for multiple projects and applications.

API User Accounts

Bytengine user accounts allow the administrator to control server and database access. A session key is created on user login and should be included in subsequent server protected api calls. User accounts can be granted access to more than one database.

Server Commands

Bytengine server commands have a classic shell like syntax which will be familiar to Unix/Linux users. Commands and their options are available in the commands section of the documentation. Command help is also available from Byterminal by typing 'help' or 'help *'

Files & Directories

Bytengine uses a hierarchical data storage format similar to Unix/Linux where user content is saved in Files which are further organized into Directories with a top level root ('/') directory. Files are dual layered data containers with a searchable JSON part and a binary part which allows users to 'attach' regular picture, music and pdf files among others.

Direct Access

Bytengine API is accessed via authenticated Http POST method calls. However content can be served directly by Http GET making Bytengine an ideal static file/data server. Bytengine Files must be marked as 'public' to allow them be accessed in this 'direct' manner. Further explanation can be obtained from the commands documentation.
